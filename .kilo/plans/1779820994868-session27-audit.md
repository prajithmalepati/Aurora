## BLOCKER

### B1. AuroraColorBridge never wired — `useAuroraColor` causes full App re-renders
**File:** `frontend/src/App.tsx:34`, `frontend/src/components/aurora/AuroraColorBridge.tsx:1-7`
**What:** `AuroraColorBridge` exists as a null component (per Task 3.3 spec) but is never imported or rendered in `App.tsx`. Instead, `useAuroraColor()` is called directly at `App.tsx:34`. Every song change triggers `setColor2Linear` inside the hook → full `<App>` re-render cascades to all children. The entire purpose of the null-component isolation pattern (re-render in isolation, no cascade) is defeated.
**Fix:** Render `<AuroraColorBridge />` in `App.tsx` before `<AppShell>`, remove the `useAuroraColor()` call from App, and pass `color2LinearRgb` via Zustand store instead of React state.

### B2. WaveformBar directly accesses Howler.js internals via `window`
**File:** `frontend/src/components/player/WaveformBar.tsx:53`
**What:** `(window as any).Howler?._howls?.[0]` — bypasses both Zustand playerStore and the `howlRef` from `useAudioPlayer`. During crossfade, `_howls` contains multiple Howls; `_howls[0]` is not guaranteed to be the currently playing one. The seek position read (`howl.seek()`) may return the wrong song's position, causing the playhead to jump erratically.
**Fix:** Expose the active `Howl` via `playerStore` (store the Howl instance returned by `useAudioPlayer`), or pass `seek`/`progress` from PlayerBar as props since PlayerBar already has `seek` from `usePlayerStore`.

### B3. `draw` RAF loop continues after WebGL context lost with no recovery path
**File:** `frontend/src/components/aurora/AuroraCanvas.tsx:237-238`
**What:** `draw` is defined with `useCallback(() => {...}, [])`. The `onContextLost` handler cancels RAF. But `onContextRestored` then re-calls `initWebGL(canvas)` — which calls `canvas.getContext('webgl', ...)`. Per WebGL spec, after context loss, `getContext` returns **null** until the `webglcontextrestored` event fires. However, if context loss occurs and the restore event **never fires** (Firefox 16-context limit, GPU process crash), `onContextRestored` never runs, RAF remains cancelled, and the canvas permanently shows whatever the last frame was (or transparent). The fallback `.aurora-fallback` class is never activated because `webglFailed` is only set on **initial** init failure, not on context loss.
**Fix:** In `onContextLost`, also set `setWebglFailed(true)` to trigger the CSS fallback if the context is lost and not restored within a timeout.

### B4. Audio chain breaks on every song change (audio dropout)
**File:** `frontend/src/hooks/useAudioAnalyser.ts:79-86`
**What:** The effect cleanup disconnects source→analyser and analyser→destination. `createMediaElementSource` reroutes the `<audio>` element's output through the source node. Between cleanup and the next effect run's `source.connect(analyser); analyser.connect(destination)`, the source has **zero connections** → audio is silent. This happens on every song change because `currentId` is a dep. The disconnect/reconnect gap produces an audible dropout (typically 1-2 frames at 60fps but noticeable on rapid song changes).
**Fix:** Don't disconnect from destination on cleanup when the same `audioEl` is being reused. Use `createGain` as a passthrough node that's never disconnected.

### M4. `useAudioAnalyser` accesses Howler internals via `(window as any)`
**File:** `frontend/src/hooks/useAudioAnalyser.ts:33-36`
**What:** `Howler._howls[0]._sounds[0]._node` — same fragile pattern as WaveformBar (B2). If Howler hasn't initialized yet, `Howler._howls` is undefined. ## MINOR

### N1. Curtain phase offsets are decorated round numbers, not irrational
**File:** `frontend/src/components/aurora/AuroraCanvas.tsx:78-81`
**What:** HANDOFF.md describes them as "irrational phase offsets." Actual values: 0.00 (zero), 1.70 (terminating decimal), 3.14 (π to 2dp — a rational approximation), 5.30 (terminating decimal). Only π's true value is irrational — 1.70 and 5.30 are terminating decimals. The spec's claim that "they will never align" is true in practice (their ratios are not small integers), but the description is misleading.

### N2. `initWebGL` leaks GPU shaders on compile/link failure
**File:** `frontend/src/components/aurora/AuroraCanvas.tsx:102-159`
**What:** If `compileShader` fails for one shader but succeeds for the other, or if program linking fails, the function returns `null` without calling `gl.deleteShader` on objects created before the failure path. This leaks WebGLShader/WebGLProgram objects. Rare in practice (shader source is static), but a correctness issue.

### N3. Paused crossfade resumes outgoing song at partial volume
**File:** `frontend/src/hooks/useAudioPlayer.ts:192-199`
**What:** The `isPlaying` effect calls `prevHowlRef.current?.play()` on resume. If user pauses mid-crossfade, the outgoing song (`prevHowl`) was being faded to 0. On resume, it plays at its current (partially-faded) volume alongside the current song until its setTimeout fires. The outgoing song is audible again during the remainder of its fade window.
**Fix:** Check if `prev` volume is near 0 before calling `play()`.

### N4. Canvas `offsetWidth`/`offsetHeight` may be 0 on first ResizeObserver callback
**File:** `frontend/src/components/aurora/AuroraCanvas.tsx:276-278`
**What:** The ResizeObserver callback reads `canvas.offsetWidth` and `canvas.offsetHeight`. On first observation, these could be 0 if the canvas has just been inserted into a hidden or zero-size container. Setting canvas dimensions to 0 causes WebGL to produce a 0×0 framebuffer; `gl.viewport(0, 0, 0, 0)` is a no-op. The canvas renders nothing until the next resize.
**Fix:** Guard with `if (canvas.offsetWidth === 0 || canvas.offsetHeight === 0) return`.

### N5. PNG with alpha composites against black in color extraction
**File:** `backend/app/services/file_scanner.py:201`
**What:** `Image.open(...).convert("RGB")` fills transparent areas with black (default). For album art with alpha transparency (e.g., circular logos, cutout art), the black fill skews the dominant color toward dark tones.
**Fix:** Composite against a neutral background before quantizing, or use `Image.quantize` directly on RGBA data.

### N6. Handoff commit 5558557 has a body
**File:** `git log 5558557`
**What:** CLAUDE.md: "Commit messages use `type(scope): description` format only. No Co-Authored-By, **no body**, no footer." The handoff commit includes the entire session summary as a body.
